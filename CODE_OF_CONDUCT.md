# Code of Conduct

## 我们的承诺

本项目希望保持一个专业、友善、可协作的维护环境。参与讨论、提交 issue、发起 PR、评审代码或复现问题时，请默认以善意沟通，并尊重不同经验背景的参与者。

## 期望行为

- 使用清晰、具体、尊重他人的表达方式。
- 聚焦问题本身，避免人身攻击、嘲讽或贬低。
- 在提出反对意见时说明原因、影响和可选方案。
- 发现安全问题或敏感信息时，不在公开 issue 中披露细节。
- 评审代码时优先指出风险、复现方式和改进建议。

## 不接受的行为

- 骚扰、歧视、威胁、辱骂或持续冒犯。
- 故意公开他人的隐私、凭据、token、日志敏感信息或内部数据。
- 在技术讨论中反复刷屏、偏离主题或阻碍维护者处理问题。
- 明知存在安全风险仍鼓励他人公开利用细节。

## 维护者职责

维护者可以对违反本准则的内容采取编辑、隐藏、关闭、锁定、删除或限制参与等措施。处理时会尽量保持透明、具体和可复核，但涉及隐私或安全问题时可能不会公开全部细节。

## 适用范围

本准则适用于本仓库的 issue、PR、review、discussion、commit message、文档协作以及相关项目沟通渠道。
