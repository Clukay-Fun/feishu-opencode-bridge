# 劳动 Skill 输出设计

## 定位

劳动 skill 不是两个孤立功能，而是同一个业务域下的两个 workflow：

```text
劳动 skill
  ├─ 证据链分析
  └─ 基于证据链的文书起草
```

它的目标不是单独审查一份合同，而是把案件材料整理成可供律师继续工作的结构化案件认知。

合同审查应当被理解为“证据链分析”中的关键文书专项分析，而不是独立的大功能。

## 输入与输出

### 输入

优先支持以下劳动争议高频材料：

- 劳动合同 / 补充协议 / 竞业协议
- offer / 入职通知 / 通知函
- 工资单 / 工资流水 / 社保公积金材料
- 考勤记录 / 排班记录
- 聊天记录 / 邮件
- 解除通知 / 离职协议 / 交接材料
- 录音转写文本
- 仲裁申请书 / 答辩状 / 证据目录等仲裁材料

### 输出

默认输出一个飞书文档。

该文档不是“纯文字报告”，而是一个可读、可扫、可继续编辑的案件工作文档。

建议风格：

- 顶部一屏给出结论
- 中部用表格承载证据链和风险
- 只在关键处使用画板
- 底部保留待补材料和下一步动作

## 知识库作用

劳动 skill 中，知识库不负责原始材料读取，而负责法律意义解释与争议判断。

建议按以下方式接入：

- 在证据链分析里，知识库负责解释证据的法律意义、支持方向、争议点与风险。
- 在文书起草里，知识库负责补充法律依据、组织请求项与论证逻辑。
- 当用户直接要求起草文书时，内部仍应先完成一轮证据链整理，再进入草稿生成。

## 飞书文档结构

建议默认输出以下结构。

### 1. 案件摘要

这一节控制在文档首屏，像总览卡片。

建议字段：

- 案件主题
- 生成时间
- 材料数量
- 当前争议阶段
- 当前结论
- 风险等级
- 当前建议动作

### 2. 核心判断

用短段落说明：

- 当前主要争议焦点
- 哪些主张证据较强
- 哪些主张证据不足
- 当前最需要补的材料

这一节应当像律师给同事或客户的执行摘要。

### 3. 证据链总表

这是主体表格，建议固定列：

| 证据名称 | 类型 | 证明事实 | 支持方向 | 证明力 | 风险/缺口 | 备注 |
|----------|------|----------|----------|--------|-----------|------|

设计目标：

- 一眼看出证据链是否闭环
- 一眼看出每份材料支持哪类主张
- 一眼看出当前缺什么

### 4. 关键事实时间线

优先展示入职、岗位变化、薪资变化、沟通节点、通知节点、解除节点、仲裁前动作。

实现方式二选一：

- 材料较少时，用飞书文档表格
- 材料较多时，插入飞书画板做时间轴

### 5. 争议焦点与风险

这一节建议拆成两块：

- 争议焦点
- 风险与待核实事项

如需专项审查合同或关键文书，可在本节补一张小表：

| 材料/条款 | 风险点 | 可能后果 | 建议处理 |
|-----------|--------|----------|----------|

### 6. 法律依据与知识库支撑

不要写成长篇法条堆叠，建议用依据表：

| 争议点 | 对应规则/法条/知识点 | 与本案关系 |
|--------|----------------------|------------|

这一节的目标是说明判断依据来自哪里，而不是替代法律意见书。

### 7. 文书草稿

仅在用户要求起草时出现。

建议先放状态提示，再放正文：

- 本节内容为初稿
- 哪些事实已确认
- 哪些事实待补充

文书类型优先收敛为：

- 仲裁申请书
- 答辩状
- 情况说明
- 证据目录

### 8. 待补材料与下一步建议

这一节固定保留。

建议列出：

- 缺失证据
- 建议补充材料
- 下一步动作
- 需要人工确认的问题

这能把文档从“分析结果”变成“案件工作台”。

## 画板与表格使用原则

### 画板

不为炫技而使用画板，只在以下两类信息里使用：

- 案件时间线
- 证据关系图

推荐关系图结构：

```text
争议焦点
  -> 关键事实
    -> 对应证据
      -> 缺失环节
```

### 表格

全篇控制在 2 到 3 张核心表格以内，优先级如下：

1. 证据链总表
2. 风险点表
3. 争议点与法律依据对应表

避免把整篇文档都写成表格，影响阅读节奏。

## 推荐页面骨架

```text
案件摘要
核心判断
证据链总表
关键事实时间线
争议焦点与风险
法律依据与知识库支撑
文书草稿（可选）
待补材料与下一步建议
```

## 演示建议

如果只做一个最小可演示版本，建议固定为：

```text
上传或指定一组劳动争议材料
  -> 输出飞书文档
    -> 文档里至少包含摘要、证据链总表、时间线、风险提示、待补材料
```

如果需要强调专项能力，可在“争议焦点与风险”中插入一段合同条款审查结果，但不单独把它包装成独立主功能。

## 实现提示

bridge 侧重点应当是把“劳动 skill 的输出结构”稳定下来。

真正写入飞书文档、表格、画板时，可在用户明确要求落地到飞书资源时，再通过 `lark-cli` 完成。
