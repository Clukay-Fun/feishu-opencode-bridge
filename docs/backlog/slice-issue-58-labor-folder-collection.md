# Slice: Issue #58 劳动材料文件夹自然语言收集

## 目标

让劳动案件工作台在材料收集态下支持两种新输入：

1. 用户用自然语言描述本地目录（如"分析桌面上的劳动案件文件夹里的文件"），自动收集目录内支持的文件
2. 飞书上传文件夹时，把同一批文件聚合到当前材料收集上下文

实现在 `LaborRuntimeModule` 内部，不动 bridge core，不通用化到其他业务。

## 范围

### 包含

- `LaborRuntimeModule` 材料收集态新增自然语言路径识别
- 飞书文件夹上传的多文件聚合（按同 conversation / sender / 时间窗口或可用的 folder metadata）
- 跳过清单：混入不支持类型不导致整批失败，在卡片或 notice 中明确提示
- 桌面候选路径推断：有限的、可测试的候选集合（如 `~/Desktop/劳动案件文件夹`、`~/Desktop/劳动案件`），同时允许用户显式提供绝对路径
- 批量收集结果反馈：成功加入数量、跳过文件清单、失败原因、当前累计材料数
- 测试覆盖自然语言目录、显式路径、飞书文件夹上传、跳过不支持文件、低置信不劫持

### 不包含

- **不动** bridge core 的会话、队列、卡片回调所有权
- **不做** 通用文件夹批处理（合同、发票、知识库不在本 slice 范围）
- **不改** 单文件上传现有流程
- **不删** slash command 兜底（`/案件工作台`、`/完成上传`、`/材料收集完成` 仍然可用）
- **不改** `mappings.json` wire shape 或任何持久化字段

### 行为规则

1. **低置信不劫持**：解析置信度不够时，不进入劳动批量分析流程，保留普通对话处理。
2. **跳过不失败**：目录里有不支持类型时，支持的文件继续收集，不支持的文件在卡片中明确列出。
3. **路径来源透明**：所有目录解析（包括桌面候选推断）都要在反馈卡片里写明实际定位到的绝对路径，避免用户以为系统抓错目录。
4. **复用合同助手验证过的路径解析**：不重复造轮子，但**不要把合同模块的代码移到 bridge core 共享层**。如果需要共享，先在 `src/labor` 下做局部封装。
5. **聚合窗口可调**：飞书文件夹上传的聚合时间窗口必须是配置项，不写死。

## 实现步骤

### 步骤 1：复读现有合同助手路径解析

阅读 `src/contract-assistant/` 下与自然语言路径解析相关的实现，识别可复用的纯函数。

不做：把这些函数迁移到 `src/runtime` 或 `src/bridge`。本 slice 内只在 `src/labor` 内部使用。

### 步骤 2：扩展 LaborRuntimeModule 材料收集态

在 `src/labor/` 的材料收集逻辑中：

- 添加自然语言目录识别：匹配"分析 ... 文件夹 ...""...目录里的文件" 这类模式
- 提取候选路径：显式绝对路径优先，没有则尝试桌面候选集合
- 调用路径解析函数（复用合同助手）拿到候选目录
- 列出目录内文件，按白名单过滤
- 加入 pending labor interaction 的材料池

### 步骤 3：飞书文件夹上传聚合

在 LaborRuntimeModule 的飞书文件消息处理里：

- 检测同一 conversation / sender / 时间窗口（可配置）内的多个文件消息
- 聚合到同一批 pending 材料
- 卡片实时更新材料数量和最近材料列表

### 步骤 4：跳过清单与反馈卡片

- 失败/跳过文件：在材料收集卡里加一个"已跳过"区块，列出文件名 + 原因（不支持类型 / 大小超限 / 解析失败）
- 成功反馈：卡片显示本批新增 N 个，累计 M 个

### 步骤 5：测试

新增测试覆盖：

- 显式绝对路径：发送 `/材料收集` 后输入绝对路径，目录内支持文件被收集
- 桌面自然语言：发送"分析桌面上的劳动案件文件夹里的文件"，能定位到候选目录
- 飞书文件夹上传：多文件消息聚合到同一 pending
- 跳过不支持类型：混入 `.exe`，反馈卡显示跳过清单
- 低置信不劫持：发送"分析一下这个案子"（无路径线索），不进入批量分析

## 给执行 Agent 的硬约束

1. **业务领域代码必须留在 `src/labor/`**。即使发现合同助手的路径解析可以复用到 bridge core 共享层，本 slice 内禁止迁移。如果共享需求强，记一条 follow-up，不在本 slice 内动手。
2. **不改 `LaborRuntimeModule` 之外的接口**。`saveSessionWindow`、`getSessionWindow`、`createAndBindSession` 等签名不动。
3. **不通用化**。"自然语言文件夹收集"是劳动模块的具体能力，不是通用能力。本 slice 不允许把它做成通用接口对外暴露。
4. **路径推断要克制**。桌面候选集合不超过 5 个具体路径名。不要做"AI 猜路径"。
5. **不引入新依赖**。能用 Node fs/path 实现的不引第三方包。
