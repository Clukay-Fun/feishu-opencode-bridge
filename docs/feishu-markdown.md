# 飞书输出 Markdown 语法规范

> 适用于 `feishu-opencode-bridge` 所有卡片 body 和 Plain Post 的 Markdown 内容。

## 一、支持的语法

### 文字样式

| 场景 | 语法 | 示例 | 备注 |
|------|------|------|------|
| 强调关键词 | `**粗体**` | **已切换** | |
| 命令名 | `` `反引号` `` | `/status` | |
| 代码片段 | `` `反引号` `` | `rm -rf dist/` | |
| 代码块 | ` ``` ` 包裹 | 多行命令 | 推荐加语言标注 |
| 链接 | `[文字](url)` | [opencode.ai](https://opencode.ai) | |
| 已失效内容 | `~~删除线~~` | ~~旧会话~~ | |

### 禁用样式

| 语法 | 原因 |
|------|------|
| `*斜体*` | 飞书渲染不稳定 |
| `_下划线_` | 飞书渲染不稳定 |

### 标题

```markdown
# 一级标题
## 二级标题
### 三级标题
```

- `Command Card` 区块标题用 `##`
- `Notice Card` 不用标题
- `Process Card` 视内容长度决定，长输出可用 `##` 分段

### 列表

有序列表只用于步骤或编号选择：

```markdown
1. 先做这个
2. 再做这个
```

无序列表用于并列项，统一用 `-`，不用 `*`：

```markdown
- 项目 A
- 项目 B
```

会话列表固定格式：

```text
#1  会话名称  ← 当前
#2  会话名称  04-06 15:10
```

嵌套列表最多一层，不要更深。

### 代码

行内代码：`` `代码` ``

代码块推荐带语言标注：

````markdown
```bash
npm run dev
```
````

### 链接

统一用 `[显示文字](url)`，不裸贴 URL。

### 引用

用于原始命令回显或上下文说明：

```markdown
> 引用内容
```

### 表格

用于 `/models` 等多字段列表：

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| A   | B   | C   |
```

- 不支持对齐语法（`:---:` / `---:` 无效），统一左对齐
- 列数不宜超过 4 列，移动端显示受限

### 分隔线

`Command Card` 和 `Process Card` 区块之间用 `---`，`Notice Card` 不用。

## 二、飞书不支持的语法

以下语法在飞书卡片 Markdown 中不渲染或行为异常，bridge 输出中一律禁止使用：

| 语法 | 说明 |
|------|------|
| HTML 标签 | `<details>`、`<summary>`、`<br>` 等会被忽略 |
| 图片 `![alt](url)` | 卡片 Markdown 不渲染，需用 `img` 组件 |
| 脚注 `[^1]` | 不支持 |
| 任务列表 `- [ ]` | 不支持 |
| 表格对齐 `:---:` | 不支持，只有左对齐 |
| `*斜体*` / `_下划线_` | 渲染不稳定，禁用 |

## 三、Emoji 使用规范

场景与 emoji 固定映射，不随意替换：

| 场景 | Emoji | 用于 |
|------|-------|------|
| 成功 / 完成 | ✅ | 任务完成、操作成功 |
| 失败 / 错误 | ❌ | 错误提示、操作失败 |
| 警告 / 提醒 | ⚠️ | 过期、需要注意 |
| 信息 / 提示 | ℹ️ | 中性提示、空状态 |
| 权限 | 🔐 | 权限请求 |
| 当前选中 | ← | 会话列表当前项 |
| 正在处理 | ⏳ | 任务进行中 |
| 中止 | ⏹ | 任务已中止 |

原则：

- 一个场景一个 emoji，不混用
- `Notice Card` header 不放 emoji（header 已有图标），body 第一个字符可放 emoji
- `Plain Post` 可在行首放 emoji 作为视觉提示
- 不在正文中间插 emoji

## 四、长度控制

| 位置 | 限制 | 处理 |
|------|------|------|
| Notice Card body | 1-2 行 | 超出则考虑升级为 Command Card |
| Command Card body | 无硬限制 | 内容过长时用 `##` 分段 |
| Process Card body | 单条卡片上限 30,000 字符 | 超出时截断，末尾追加 `⚠️ 输出过长，已截断` |
| Plain Post | 极短 | 超过 3 行应考虑是否该用卡片 |

## 五、转义规范

飞书 Markdown 对特殊字符的处理与标准 Markdown 不同：

| 字符 | 处理方式 |
|------|---------|
| `<` `>` | 包在反引号内：`` `<provider/model>` `` |
| `&` | 代码上下文中用反引号包裹 |
| `\|` | 表格内需转义为 `\|` |
| `` ` `` | 嵌套时用双反引号 `` `` ` `` `` |

原则：凡是出现在代码、命令、路径中的特殊字符，一律用反引号包裹。

## 六、空行与间距

- 段落间一个空行，不多不少
- 连续空行在飞书中可能被合并或产生异常间距
- 列表项之间不加空行
- 代码块前后各留一个空行

## 七、语气规范

| 规则 | 示例 |
|------|------|
| 不用感叹号 | ✅ `已切换到会话 #2` / ❌ `已切换到会话 #2！` |
| 不用“抱歉” | ✅ `服务不可用，请检查连接` / ❌ `抱歉，服务不可用` |
| 不用“请注意” | ✅ 直接说明情况 / ❌ `请注意，会话已过期` |
| 陈述事实，不解释原因 | ✅ `当前没有正在执行的任务` / ❌ `因为没有任务在执行，所以无法中止` |
| 命令名用反引号 | ✅ `请发送 \`/sessions\`` / ❌ `请发送 /sessions` |

## 八、各卡片类型的 Markdown 用法速查

### Command Card

```markdown
## 会话列表

#1  重构登录模块  ← 当前
#2  修复 CI 流水线  04-06 15:10
#3  API 文档更新  04-05 09:30

---

发送 `/switch <编号>` 切换会话
```

### Notice Card

```markdown
ℹ️ 当前没有待确认的权限请求。
```

```markdown
⚠️ 会话列表已过期，请重新发送 `/sessions`
```

```markdown
❌ OpenCode 服务不可用，请检查连接。
```

### Plain Post

```text
Session abc123 created.
Model: claude-sonnet-4-20250514
```

`Plain Post` 只用于 passthrough 文本输出、极短确认语、以及卡片不可用时的降级兜底；bridge 自有命令、结构化列表、系统提示一律不使用 `Plain Post`。

### Process Card

正常对话输出遵循上述所有语法规范。长输出用 `##` 分段，代码块带语言标注。
