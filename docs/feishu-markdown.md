# 飞书输出 Markdown 语法规范

> 适用于 `feishu-opencode-bridge` 所有卡片 body 和 Plain Post 的 Markdown 内容。

## 一、支持的语法

### 文字样式

| 场景 | 语法 | 示例 | 备注 |
|------|------|------|------|
| 强调关键词 | `**粗体**` | **已切换** | |
| 命令名 | `` `反引号` `` | `/status` | |
| 代码片段 | `` `反引号` `` | `rm -rf dist/` | |
| 代码块 | ` ``` ` 包裹 | 多行命令 | 推荐加语言标注 |
| 链接 | `[文字](url)` | [opencode.ai](https://opencode.ai) | |
| 已失效内容 | `~~删除线~~` | ~~旧会话~~ | |

### 禁用样式

| 语法 | 原因 |
|------|------|
| `*斜体*` | 飞书渲染不稳定 |
| `_下划线_` | 飞书渲染不稳定 |

### 标题

```markdown
# 一级标题
## 二级标题
### 三级标题
```

- `Command Card` 区块标题用 `##`
- `Notice Card` 不用标题
- `Process Card` 视内容长度决定，长输出可用 `##` 分段

### 列表

有序列表只用于步骤或编号选择：

```markdown
1. 先做这个
2. 再做这个
```

无序列表用于并列项，统一用 `-`，不用 `*`：

```markdown
- 项目 A
- 项目 B
```

会话列表固定格式：

```text
#1  会话名称  ← 当前
#2  会话名称  04-06 15:10
```

嵌套列表最多一层，不要更深。

### 代码

行内代码：`` `代码` ``

代码块推荐带语言标注：

````markdown
```bash
npm run dev
```
````

### 链接

统一用 `[显示文字](url)`，不裸贴 URL。

### 引用

用于原始命令回显或上下文说明：

```markdown
> 引用内容
```

### 表格

用于 `/models` 等多字段列表：

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| A   | B   | C   |
```

- 不支持对齐语法（`:---:` / `---:` 无效），统一左对齐
- 列数不宜超过 4 列，移动端显示受限

### 分隔线

`Command Card` 和 `Process Card` 区块之间用 `---`，`Notice Card` 不用。

## 二、飞书不支持的语法

以下语法在飞书卡片 Markdown 中不渲染或行为异常，bridge 输出中一律禁止使用：

| 语法 | 说明 |
|------|------|
| HTML 标签 | `<details>`、`<summary>`、`<br>` 等会被忽略 |
| 图片 `![alt](url)` | 卡片 Markdown 不渲染，需用 `img` 组件 |
| 脚注 `[^1]` | 不支持 |
| 任务列表 `- [ ]` | 不支持 |
| 表格对齐 `:---:` | 不支持，只有左对齐 |
| `*斜体*` / `_下划线_` | 渲染不稳定，禁用 |

## 三、Emoji 使用规范

> Emoji 是状态指示器，不是装饰。

### 规则

#### 1. 一对一绑定

一个语义只绑一个 emoji，一个 emoji 只表达一个语义。不允许同一个 emoji 在不同场景表达不同含义。

#### 2. 只出现在卡片 body / subtitle 的文本内容里

- `Notice Card` body 行首第一个字符
- `Process Card` subtitle
- `Command Card` body 内的状态标记

不出现在：

- 卡片 header title（header 已有 `standard_icon` + `template` 颜色承担语义）
- 正文中间（不在句中插入 emoji）
- `Plain Post`（纯文本不带 emoji，除非降级兜底需要最低限度的状态提示）

#### 3. 映射表穷举，表外不用

| Emoji | 唯一语义 | 典型出现位置 |
|-------|---------|-------------|
| ⚙️ | 处理中 | Process Card subtitle |
| ✅ | 成功 / 完成 | Process Card subtitle, Notice body |
| ❌ | 错误 / 失败 | Notice body, Process Card subtitle |
| ⚠️ | 警告 / 提醒 | Notice body |
| ℹ️ | 信息 / 空状态 | Notice body |
| 🔐 | 权限请求 | Notice body |
| ⏹ | 中止 | Notice body |
| ⏱ | 耗时 | Process Card subtitle |
| ⏳ | 队列等待 | Process Card subtitle |

不使用 `🎉` `🚀` `👋` `💬` `👤` `🚪` `🤖` `📋` 等，保持运维面板调性。

### 示例

```text
ℹ️ 当前没有待确认的权限请求。
⚠️ 会话列表已过期，请重新发送 `/sessions`
❌ OpenCode 服务不可用，请检查连接。
🔐 OpenCode 想执行：`rm -rf dist/`
⏹ 当前任务已中止，可发送新消息继续对话。
```

```text
⚙️ 处理中…
✅ 完成 · ⏱ 12.3s
❌ 失败
```

```text
#1  重构登录模块  ← 当前
#2  修复 CI 流水线  04-06 15:10
```

## 四、长度控制

| 位置 | 限制 | 处理 |
|------|------|------|
| Notice Card body | 1-2 行 | 超出则考虑升级为 Command Card |
| Command Card body | 无硬限制 | 内容过长时用 `##` 分段 |
| Process Card body | 单条卡片上限 30,000 字符 | 超出时截断，末尾追加 `⚠️ 输出过长，已截断` |
| Plain Post | 极短 | 超过 3 行应考虑是否该用卡片 |

## 五、转义规范

飞书 Markdown 对特殊字符的处理与标准 Markdown 不同：

| 字符 | 处理方式 |
|------|---------|
| `<` `>` | 包在反引号内：`` `<provider/model>` `` |
| `&` | 代码上下文中用反引号包裹 |
| `\|` | 表格内需转义为 `\|` |
| `` ` `` | 嵌套时用双反引号 `` `` ` `` `` |

原则：凡是出现在代码、命令、路径中的特殊字符，一律用反引号包裹。

## 六、空行与间距

- 段落间一个空行，不多不少
- 连续空行在飞书中可能被合并或产生异常间距
- 列表项之间不加空行
- 代码块前后各留一个空行

## 七、语气规范

> 运维面板调性：克制、准确、不废话。

### 句式规则

| 规则 | 句式 | 示例 |
|------|------|------|
| 完成态 | `已` 开头 | 已切换、已解除绑定、已中止 |
| 进行态 | `正在` 开头 | 正在执行、正在创建会话 |
| 状态态 | `当前` 开头 | 当前无活跃会话、当前没有正在执行的任务 |
| 引导下一步 | `发送 \`/xxx\` + 动词` | 发送 `/abort` 中止任务 |

| 规则 | 示例 |
|------|------|
| 不用感叹号 | ✅ `已切换到会话 #2` / ❌ `已切换到会话 #2！` |
| 不用“抱歉” | ✅ `服务不可用，请检查连接` / ❌ `抱歉，服务不可用` |
| 不用“请注意” | ✅ 直接说明情况 / ❌ `请注意，会话已过期` |
| 陈述事实，不解释原因 | ✅ `当前没有正在执行的任务` / ❌ `因为没有任务在执行，所以无法中止` |
| 不用“可以” | ✅ `发送 \`/sessions\` 查看列表` / ❌ `发送 /sessions 可以查看列表` |
| 命令名用反引号 | ✅ `请发送 \`/sessions\`` / ❌ `请发送 /sessions` |

### 各卡片类型语气示例

#### Notice Card

```text
ℹ️ 当前没有待确认的权限请求。
⚠️ 会话列表已过期，请重新发送 `/sessions`
❌ OpenCode 服务不可用，请检查连接。
⏹ 当前任务已中止，可发送新消息继续对话。
```

#### Command Card

```text
已切换到会话 #2
已创建新会话 #3
已解除绑定，后续需 @bot 重新绑定。
```

#### Process Card subtitle

```text
⚙️ 处理中…
✅ 完成 · ⏱ 12.3s
❌ 失败
```

#### Plain Post

```text
Session abc123 created.
已确认权限请求。
```

## 八、各卡片类型的 Markdown 用法速查

### Command Card

```markdown
## 会话列表

#1  重构登录模块  ← 当前
#2  修复 CI 流水线  04-06 15:10
#3  API 文档更新  04-05 09:30

---

发送 `/switch <编号>` 切换会话
```

### Notice Card

```markdown
ℹ️ 当前没有待确认的权限请求。
```

```markdown
⚠️ 会话列表已过期，请重新发送 `/sessions`
```

```markdown
❌ OpenCode 服务不可用，请检查连接。
```

### Plain Post

```text
Session abc123 created.
Model: claude-sonnet-4-20250514
```

`Plain Post` 只用于 passthrough 文本输出、极短确认语、以及卡片不可用时的降级兜底；bridge 自有命令、结构化列表、系统提示一律不使用 `Plain Post`。

### Process Card

正常对话输出遵循上述所有语法规范。长输出用 `##` 分段，代码块带语言标注。
