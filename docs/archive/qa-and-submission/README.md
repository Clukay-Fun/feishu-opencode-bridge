# 提交与冻结归档说明

这个目录只保留“提交阶段或框架冻结阶段仍有里程碑价值”的历史资料。

它的定位不是保存所有提交期文档，而是保留少量仍能回答下面问题的记录：

- 框架冻结当时的验收结论是什么
- 当时用什么证据确认边界已经收紧
- 哪些准入要求是在冻结阶段被明确下来的

如果你想看项目现在应该怎么开发，优先看这些现行文档：

- `AGENTS.md`
- `docs/architecture-baseline.md`
- `docs/guidelines/new-feature-checklist.md`
- `README.md`

## 当前保留资料

### `freeze-acceptance.md`

这是一份框架冻结验收记录。

- 用途：记录冻结阶段的验收范围、证据采样和通过判断
- 为什么保留：它是一个历史里程碑，能解释“框架冻结”不是口头说法，而是经过检查和确认的状态

### `2026-04-24-issue-completion-report.md`

这是一份 issue 收口完成报告。

- 用途：汇总 2026-04-23 至 2026-04-24 期间完成关闭的 issue，并整理发布前回归测试清单
- 为什么保留：它记录了框架去业务化、文档解析、劳动工作流、知识库一致性和短期上下文等一组连续收口工作的验收入口

## 保留标准

以后只有满足下面条件的资料，才适合继续留在这个目录：

- 它记录了一个重要历史里程碑
- 它能解释当前仍然有效的验收结论或准入边界
- 它提供的是长期背景，而不是短期执行步骤

下面这些通常不建议保留：

- 提交 checklist
- PR 拆分计划
- 临时执行手册
- 只服务某次提交窗口的施工文档

## 使用原则

- 这里的文档只提供历史背景，不代表当前最新规则
- 当前规则以 `AGENTS.md`、`docs/architecture-baseline.md` 和 active guidelines 为准
- 如果归档文档与现行规则冲突，以现行规则为准
