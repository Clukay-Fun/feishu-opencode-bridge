# 2026-04-10 人工验收结果总结

## 结论

本轮按提交版主线完成了私聊、群聊、会话管理、模型查看、权限 fallback、启动检查的人工验收。

当前结论为：

- 核心链路通过
- 提交版可继续推进
- 个别能力已明确边界，不计入当前版本缺陷

## 已通过范围

### B1 私聊新建与当前状态

- `/new` 正常创建新会话
- 普通消息正常进入 Process Card 并完成回复
- `/status` 正常返回会话状态卡

### B2 会话列表与切换 / 删除

- `/sessions`
- `/switch <编号>`
- `/close`
- `/delete`
- `/close all`
- `/delete all`
- `/close <起始-结束>`
- `/delete <起始-结束> confirm`
- `/sessions all`

以上能力均已通过真机验证。

补充结论：

- `/close` 为软删除，只从 bridge 当前窗口移除
- `/delete` 为硬删除，会同时删除 OpenCode 真实 session
- `/sessions all` 已改为展示全部会话，并按多张卡分页，避免飞书单卡元素超限

### B3 模型查看

- `/model`
- `/model <provider>`

已通过真机验证，当前均为卡片输出。

补充结论：

- `/model use ...`
- `/model reset`

当前不属于已实现能力范围，不计为本轮缺陷，详见 GitHub issue：

- [#9 明确模型命令边界并评估窗口级切换方案](https://github.com/Clukay-Fun/feishu-opencode-bridge/issues/9)

### C 群聊绑定闭环

以下场景均已通过真机验证：

- 未绑定用户未 `@bot` 发消息会被静默忽略
- 首次 `@bot` 普通消息可完成绑定
- 绑定后可免 `@bot` 连续对话
- `/who` 正常返回绑定状态卡
- `/leave` 正常解绑
- 解绑后再次未 `@bot` 发消息会被静默忽略

### D 权限确认

当前本地环境按文本 fallback 验收通过。

补充结论：

- 当前 `config.json` 未启用 `feishu.cardActions.enabled`
- 因此权限请求按预期走文本命令 fallback：
  - `/allow once`
  - `/allow always`
  - `/deny`
- 本地环境未按“真按钮”验收

### E 启动与 preflight

以下场景已通过终端验收：

- `opencode serve` 正常时，`npm run dev:once` 可通过 preflight 并成功启动
- `opencode serve` 停止时，preflight 会在 OpenCode 健康检查处明确失败并退出
- 数据目录与日志目录可写检查通过
