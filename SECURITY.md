# Security Policy

## 支持范围

当前项目仍处于快速维护阶段，安全修复默认只面向 `main` 分支最新版本。历史版本通常不会单独回补，除非维护者明确说明。

## 报告安全问题

如果你发现安全问题，请不要在公开 issue 中披露可利用细节、凭据、日志片段或真实用户数据。

推荐处理方式：

1. 优先使用 GitHub 的 private vulnerability reporting 或 Security Advisory 功能提交报告。
2. 如果仓库未开启私密报告，请通过维护者已知的私密沟通渠道联系，并只提供最小必要复现信息。
3. 在维护者确认前，请不要公开 PoC、攻击路径、token、日志或受影响数据。

## 报告内容建议

请尽量包含：

- 受影响的功能或文件路径。
- 复现步骤和触发条件。
- 可能影响的数据、权限或外部系统。
- 你已验证的版本、commit 或运行环境。
- 是否已经发现公开利用迹象。

## 处理预期

维护者会尽快确认报告是否有效，并评估影响范围、修复方案和披露节奏。复杂问题可能需要更多时间复现、补测试和发布修复。

## 敏感数据

本项目可能接触 Feishu/Lark app 凭据、OpenCode 会话、合同材料、知识库数据和本地日志。提交 issue、PR 或示例配置时，请先脱敏：

- 删除 token、secret、cookie、authorization header。
- 删除真实姓名、手机号、邮箱、合同编号、聊天记录等个人或业务敏感信息。
- 使用最小化 mock 数据复现问题。
