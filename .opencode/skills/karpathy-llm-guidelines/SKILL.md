# Karpathy LLM 协作指南

本技能源于 Andrej Karpathy 的 LLM 使用洞见，适配 OpenCode 交互场景。

## 核心原则

### 1. Think Before You Code

在写代码前，先思考问题本身。这有助于生成更高效的提示词。

- **解构问题**：将复杂问题拆分为更小、更易管理的部分
- **明确目标**：在提示词中尽可能具体地描述期望结果
- **一次一事**：专注于当前任务的一个部分，避免多目标混杂

### 2. Be Specific & Provide Context

提示词越具体，LLM 越能理解你的需求并生成正确代码。

- **明确编程语言、框架、库版本**等上下文
- **提供相关代码片段**，便于 LLM 理解现有代码风格
- **使用清晰简洁的语言**，避免歧义
- **指定输出格式**，如"用 TypeScript 实现"、"返回 JSON 格式"

### 3. Iterate and Refine

不要期望一次成功。准备好迭代和优化提示词。

- **从简单提示词开始**，逐步添加细节
- **分析输出**：如果不符合预期，分析差距并调整提示词
- **分步验证**：复杂任务分步骤进行，每步验证后再继续
- **提供反馈**：如果代码有问题，明确指出问题所在

### 4. You are the expert

你才是领域专家。AI 是工具，始终保持批判性思维。

- **审查每一行代码**：不要假设 AI 生成的代码一定正确
- **测试验证**：运行代码确认行为符合预期
- **安全意识**：注意代码中的安全漏洞、隐私问题
- **按需修改**：不要害怕调整 AI 生成的代码以满足需求

## OpenCode 专项场景

### 高效提问模板

```
## 任务
[具体描述你要做什么]

## 上下文
- 语言/框架: [技术栈]
- 相关文件: [文件路径或关键代码]
- 约束条件: [性能/安全/兼容性要求]

## 期望输出
[具体描述期望结果]
```

### 代码修改场景

```
[粘贴相关代码片段]

请审查上述代码，重点关注：
1. [具体问题1]
2. [具体问题2]

期望修改为：
[描述期望行为]
```

### Debug 场景

```
错误信息：
[粘贴错误日志]

相关代码：
[粘贴相关代码]

已尝试的排查步骤：
1. [步骤1]
2. [步骤2]
```

## 常见陷阱

| 陷阱 | 应对方法 |
|------|----------|
| 提示词过于模糊 | 添加具体约束条件和示例 |
| 上下文不足 | 提供相关代码和文件路径 |
| 一次要求太多 | 拆分为多个独立任务 |
| 不验证输出 | 始终测试和审查生成结果 |
| 假设 AI 全知 | 明确指出你的具体需求和约束 |
