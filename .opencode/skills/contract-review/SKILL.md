# Contract Review

当用户要审查已有合同、识别风险、检查缺失条款或获得修改建议时，优先走这条专项 skill。

当前定位是合同工作域的“审查”能力：它可以读取材料、输出风险和建议，但默认不修改合同原文，也不写入合同台账。

## 触发

自然语言可直接触发，例如：

```text
审查一下刚才这份合同
帮我看看这个合同有什么风险
检查 /Users/me/contracts/demo.pdf 里哪些条款需要改
```

强制入口建议：

```bash
/合同审查
/contract-review
/合同审查 /absolute/path/to/contract.pdf
```

## 输出结构

- 审查结论：一句话概括整体风险等级。
- 关键风险：按严重程度列出问题、依据、影响和建议。
- 缺失条款：列出应补但没有发现的条款。
- 可直接修改建议：给出可复制到合同中的替代表述。
- 待确认问题：列出需要用户补充背景才能判断的点。

## 边界

- 不默认改写整份合同。
- 不默认写入合同台账。
- 如果用户要求“按建议直接改”，应转入 `contract-revise`。
- 如果用户要求“提取字段并入表”，应转入 `contract-extract`。
- 如果用户只是要求摘要，不应强行输出法律审查报告。
